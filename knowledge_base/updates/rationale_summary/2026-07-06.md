# Rationale Summary: 2026-07-06

## Dataset Prioritization

本轮优先固化通用 benchmark，因为它们对 OPF、PF、N-1、控制、预测和图学习任务复用率最高，且访问成本低。MATPOWER、pandapower networks、PGLib-OPF、RTS-GMLC、SimBench 和 Grid2Op 能覆盖从静态网架到动态运行控制的主要实验骨架。

## Metadata-Only Sources

EIA、NREL NSRDB、ENTSO-E、PJM、ACN-Data 等平台通常需要 API key、token、注册或明确 endpoint 选择。Zenodo large synthetic power grid ML dataset 体量约 TB 级，默认全量下载不适合作为项目初始化动作。因此本轮保存入口、元数据和后续脚本策略。

## Literature Scope

近三年文献筛选优先考虑 IEEE/SCI/高影响数据论文和 benchmark 论文。当前知识库先建立“任务-数据集-baseline-文献”的骨架，后续每篇进入 ARA 时再补齐 BibTeX、PDF、代码和复现实验。
