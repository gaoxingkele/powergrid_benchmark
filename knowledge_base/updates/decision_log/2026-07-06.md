# Decision Log: 2026-07-06

## Decisions

1. 建立 `knowledge_base/` 作为项目全局知识库，独立于单篇论文 ARA。
2. 将公开数据集分成九类：标准网架、OPF benchmark、生产成本/可靠性、配电网、RL 控制、真实时序/市场、天气/可再生、EV/用户侧、设备/PMU/故障。
3. 先下载小到中等体量且可直接 clone 的核心数据：MATPOWER、pandapower networks、PGLib-OPF、RTS-GMLC、SimBench、Grid2Op datasets index。
4. 对 OPSD、EIA、NREL NSRDB、ACN-Data、TAMU、Zenodo TB 级数据先保存元数据或入口页。
5. “思维链更新”改为 `decision_log`、`rationale_summary`、`open_questions` 三类可审计记录，不保存隐藏推理。
6. 继续推进“全量本地缓存”时，将数据源分为 direct-download、git-cache、metadata-only/API-token、TB-scale 四类；能自动缓存的立即落地，需要凭证或 TB 级的数据保留本地入口和脚本策略。

## Consequences

- 后续论文工程优先引用全局 dataset registry 和 baseline matrix。
- 大体量/API 数据不会被误认为已经完整下载。
- 每一轮知识更迭有固定文件入口，方便审阅和回滚。
